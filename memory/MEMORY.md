# EEZ Agent Memory

Operational memory loaded at session start. These files capture confirmed patterns, recurring failures, and calibrated heuristics — not static knowledge. Update entries when a new pattern is confirmed or a prior one is invalidated.

## Index

- [EEZ Voice Patterns](eez-voice-patterns.md) — recurring voice corrections and confirmed rules for EEZ content
- [Technical Accuracy Watchlist](technical-accuracy-watchlist.md) — five distinctions that consistently get blurred; fast-load reference before publishing
- [Partner Engagement Patterns](partner-engagement-patterns.md) — confirmed outreach patterns per mode and partner type

## Usage

Load all three files before any MEDIUM+ task involving partner content, external copy, or technical claims. On session start, scan for entries marked `[UPDATED]` to catch recent calibrations.

## Meta

Memory is owned by the eez-agent system. Updates happen here, not in `knowledge/`. Static reference material lives in `knowledge/eez/` and `knowledge/reference/`.
