# EEZ Communications Framework

**Purpose:** the canonical messaging framework for EEZ. Used by the Gnosis team, alliance partners, and external amplifiers. Partners reshare these pillars in their own words. The strategy is that partner-led amplification carries more weight to a technical audience than self-promotion.

**Tone target:** exclusive, underground, builder-credible. No hype. No project self-promotion language. Statements of position on Ethereum's future.

---

## The five messaging pillars

### 01. The Problem (Why Now)

The anchor pillar. Every partner should be able to articulate what is broken in their own words.

**Core narrative.** Ethereum's L2 security promise has become misleading. Liquidity is fragmented across rollups. Security guarantees are not being delivered. Economic value leaks out of Ethereum to private operators. This is not a complaint about any specific L2; it is a statement about Ethereum's structural position.

**Use this pillar when:** a partner is taking a public stance on Ethereum fragmentation, the L2 value extraction debate, or where Ethereum needs to go next.

**Example framing:**
> "Ethereum's security underwrites a trillion-dollar ecosystem. It deserves rollups that actually inherit it."

---

### 02. The Design Principle (Native, Not Hosted)

The technical credibility pillar. The sharpest narrative edge.

**Core narrative.** There is a difference between rollups built *on* Ethereum and rollups built *by* Ethereum. EEZ is the second kind. Same governance, same client diversity, same security culture. No multisigs, no upgrade keys, no admin backdoors.

**Use this pillar when:** a block builder, infra provider, or core dev team is amplifying. They are signalling alignment with Ethereum's values, not endorsing a project.

**Example framings:**
> "There's a difference between rollups that settle to Ethereum and rollups that are Ethereum. We're building the second kind."

> "If Ethereum can govern a $400B consensus layer, it can govern a rollup. The question was never capability. It was priority."

---

### 03. Economic Realignment

The pillar that resonates most viscerally with the Ethereum community right now.

**Core narrative.** Most L2 economic activity does not use Ethereum for sequencing. Sequencing value, MEV, and fees flow to private operators rather than back to the protocol. EEZ rollups are economically aligned with Ethereum by design. Sequencing value flows back. The issuance mechanism that funds validator rewards could be extended to fund ZK proof generation as a protocol-level public good.

**Use this pillar when:** the discussion is about treasury, value capture, or Ethereum's long-term economic relevance. Partners can take a public position here without naming EEZ.

**Crucial framing note:** this is not a criticism of existing L2 teams. It is an observation about what Ethereum's *own* commodity rollup layer should look like.

---

### 04. The Composability Unlock

The builder-facing pillar.

**Core narrative.** Synchronous composability across rollup instances. A contract on one rollup can call a contract on another and get the result back in the same transaction. No bridges, no async messaging, just composability. Real-time ZK proving makes this possible.

**Use this pillar when:** the audience is application developers, DeFi protocols, or anyone who has built across multiple L2s and felt the pain.

**Example framings:**
> "A contract on one rollup calling a contract on another, and getting the result back in the same tx. No bridges, no async messaging, just composability. Real-time ZK proving makes this possible, and it's core to what we're building with EEZ."

> "Liquidity stops behaving like separate markets and starts behaving like a single market. Everything stays where it is."

---

### 05. The Alliance Model (We, Not It)

The meta-pillar. About *how* EEZ is being built.

**Core narrative.** No single team owns this. No foundation operates it. EEZ is being shaped by a small group of aligned builders who decided to do something about a problem everyone talks about but nobody owns. The exclusive, underground feel is real, not manufactured: this is private to builders right now.

**Use this pillar when:** introducing the alliance, recruiting new members, or explaining why this is different from a standard product launch.

## Partner-specific framings

| Partner type | Frame around |
|---|---|
| Block builders | Governance: "The same EIP process, the same core dev calls, the same deliberation. If it's going to be Ethereum's rollup, it should be governed like Ethereum." |
| Core infra teams | Trust assumptions: "No multisigs. No upgrade keys. No admin backdoors. The trust model is the same as L1." |
| Security-focused orgs | Assurance: "It's a launch requirement. That's the standard Ethereum demands." |
| L2 ecosystem teams | Complementarity: "EEZ is the commodity layer. L2s are the innovation layer. Both are necessary. Both play different roles." |
| DeFi protocols | Composability: synchronous cross-rollup calls, single-deployment liquidity, no bridge risk. |

---

## Banned framings

Do not use these. They make EEZ sound like a standard project launch and lose the audience.

- "We're excited to announce..."
- "Game-changer", "next-generation", "revolutionary"
- "Killer of [other L2]"
- Any framing that positions EEZ against existing L2s as competitors
- Any claim that users need to migrate
- Any claim that does not have technical sign-off

---

## Reference voices

When in doubt, the EEZ voice should sound like:
- Martin Köppelmann on the Bankless episode (clear technical framing, no hype)
- Vitalik on a research post (precise, hedged where appropriate)
- A core dev on All Core Devs (matter-of-fact, builder-credible)

It should *not* sound like a standard L2 launch tweet.
